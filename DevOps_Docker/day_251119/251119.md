## Shebang (hashbang)
- 스크립트의 첫 번째 줄에 들어가는 특별한 표기로, 해당 스크립트를 실행할 해석기(인터프리터)를 지정
```
 #!/bin/bash
```

## bash 스크립트 언어
- 변수: VAR="value"로 선언, \$VAR로 참조. 위치 매개변수(\$1, $2 등)와 사용자 입력(read)도 활용 가능
- if 문: [ 조건 ]을 만족하는지 검사하여 분기
- case 문: 다중 패턴에 따른 분기 처리
- for 문: 목록 또는 범위를 순회하며 반복 실행
- while 문: 주어진 조건이 참인 동안 반복 실행
- until 문: 주어진 조건이 거짓인 동안 반복 실행

### eval 명령어는 보안 취약점을 유발할 수 있으므로 사용을 지양

위험한 예시:
```
# 사용자 입력을 eval로 실행 (매우 위험!)
read USER_INPUT
eval "$USER_INPUT"  # 악의적 명령 실행 가능: rm -rf / 등
왜 위험한가? - 사용자가 입력한 문자열을 그대로 명령어로 실행
- 명령어 주입(Command Injection) 공격에 취약 - 시스템 파괴, 데이터 유출 등의 심각한 보안 사고 가능
```

### 에러처리
```
#!/bin/bash
set -e  # 명령 실패 시 스크립트 즉시 종료

echo "작업 시작"
mkdir /tmp/mydir
cd /tmp/mydir
cp important-file.txt backup.txt  # 파일이 없으면 여기서 중단
echo "작업 완료"  # 위 명령이 실패하면 실행되지 않음
```

### 종료코드
- 0: 성공
- 1~255: 실패 (숫자에 따라 유형 다름)
```
#!/bin/bash

echo  -n "파일이름을 입력해주세요: "
read file_name

# 명령 실행
cp $file_name dest.txt

#echo "종료코드 : $?"

# 종료 코드 확인
if [ $? -eq 0 ]; then
    echo "복사 성공   종료코드 : $?"
else
    echo "복사 실패  종료코드 : $?"
    exit 1
fi
```

### 명령 성공 여부에 따른 분기
- A && B : A가 성공했을 경우만 B 실행 (mkdir /tmp/testdir && cd /tmp/testdir)
- A || B : A가 실패했을 경우만 B 실행 (ping -c 1 google.com || echo "네트워크 연결 실패")
  - 구글에 핑을 한번 날리고 실패했을 경우 "네트워크 연결 실패" 메시지 출력
