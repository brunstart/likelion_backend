## JWT
- 세션에 대한 정보를 여러 서버가 다 저장해야함
- 한 서버에 세션 정보 저장해서 접근 : 해당 서버가 문제가 생기면 큰일남
- 정보를 클라이언트가 가지면 되지 않을까라는 방법<br><br>

- 인증에 대한 정보 - 쿠키가 가지고 있으면 탈취의 위험성이 있음, 쿠키가 들어왔을 때 약속된 형태로 만들면 판단의 기준이 생길 것
- secret, signature를 변경하면 쿠키값을 들고 왔을 때 변조감지가 가능
- 근데 이 방법은 인증된 사용자 정보 자체를 가져와서 사용자인 척을 하면 대처가 안됨 (토큰의 시간이 너무 길면 안됨)
- refreshToken을 도입, 같이 씀, 서버에서 가지고 있음
