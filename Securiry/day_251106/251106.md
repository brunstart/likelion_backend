## UserDetails
- 나중에 DB에서 데이터 꺼내오면 직접 만들어야함
- PasswordEncoder로 비밀번호 인코딩을 해줘야 제대로 동작함

## requestMatchers
- 특정 페이지를 특정 권한이 있는 사용자에게만 보여줄수 있도록 설정하는 메소드
- permitAll() -> 권한 없어도 볼 수 있도록 설정
- hasRole() -> 특정 권한이 있는 사람만 볼 수 있게 설정
- hasAnyRole() -> 지정된 권한 중 하나만 있어도 볼 수 있게 설정

## 쓰레드 로컬
- 서블릿은 요청마다 쓰레드를 만들어서 쓰레드에서 요청을 처리함
- 스프링부트는 WAS 내장
- 특정 쓰레드에 바인딩 된 값을 쓸 수 있도록 하는 메커니즘
- 시큐리티에서는 시큐리티 컨텍스트 홀더가 그 역할을 함
