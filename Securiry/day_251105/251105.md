## WAS
- 스프링부트가 톰캣이라고 하는 웹 어플리케이션 서버를 내장하고 있음
- 스프링만 가지고는 웹 어플리케이션을 동작시킬 수 없음<br><br>

- 요청이 들어오면 쓰레드를 생성 -> 작업을 수행

## 쓰레드 풀
- 쓰레드 생성 자체가 자원을 많이 소모하는 편 : 미리 만들어두고 쓰자

## ThreadLocal (쓰레드로컬)
- 어떤 정보를 쓰레드로컬에 넣어놓으면 쓰레드 내 어떤 지점에서든 해당 정보를 사용이 가능

## Security
- 의존성 추가로 설정

### SecurityConfig
- @Bean, @Configuration, @EnableWebFilter 어노테이션 필요
- @Bean은 SecurityFilterChain 메소드 위에 선언
- @Configuration, @EnableWebFilter은 클래스 위에 선언

## SecurityContextHolder
- 인증된 사용자의 세부정보를 저장. ThreadLocal에 저장되어있어서 쓰레드 내 어디서든 정보 사용
- Authentication 내 Principal, Credentials, Authorities가 있음
