## AOP (Aspect Oriented Programming) 관점지향 프로그래밍
- 객체 지향 프로그래밍을 보완하는 프로그래밍 패러다임
- 부수적인 것들을 안할 수는 없고 포함 시 코드 수정이 필요
- 핵심 가치
  - 관심의 분리 : 핵심 로직과 부가기능을 분리

## 용어 정리
- Aspect : Advice + PointCut : 횡단관심사를 모듈화
- Advice : 횡단 관심사를 구현한 실제코드, 특정 JoinPoint에서 실행
- JoinPoint : Advice가 적용될 수 있는 모든 지점 (메소드 호출, 예외 발생 등)
- PointCut : 실제로 적용될 지점을 지정
  ```
  @Before("execution(* org.example.aopexam.exam.*Service.*(..))")
  ```
- Target : Advice가 적용될 대상 객체 (핵심 비즈니스 로직을 담은 객체)
- Weaving : Aspect를 대상 객체에 연결하려 Proxy를 생성하는 과정
